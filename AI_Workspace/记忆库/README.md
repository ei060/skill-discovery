# 记忆库 - 三层记忆架构

> **核心理念**：从事件到知识到规则，让记忆变成可检索、可复用、可进化的知识资产

---

## 📁 架构概览

```
记忆库/
├── 情景记忆/    # 具体事件和对话（按月归档）
├── 语义记忆/    # 提炼的知识和规律
└── 强制规则/    # 必须遵守的行为约束
```

---

## 🎯 三层记忆的作用

### 1. 情景记忆（Episodic Memory）
**作用**：记录具体事件和对话的完整过程

**价值**：
- 让你能回溯 "当时怎么想的"
- 保留完整的上下文信息
- 为后续提炼提供原始素材

**命名规范**：`MMDD-简短描述.md`

**示例**：
```
情景记忆/2026-02/
├── 0213-讨论爆款标题.md
├── 0215-系统架构设计.md
└── 0220-AI工作流优化.md
```

---

### 2. 语义记忆（Semantic Memory）
**作用**：提炼可复用的通用知识和规律

**价值**：
- 知识可以复用，不用每次重新讨论
- 形成方法论和模式库
- 跨场景应用

**命名规范**：`[主题]_[类型].md`

**示例**：
```
语义记忆/
├── 爆款标题方法论.md
├── 系统架构设计模式.md
├── AI_Roland_v2_系统架构.md
└── 内容创作SOP.md
```

---

### 3. 强制规则（Procedural Memory）
**作用**：必须遵守的行为约束

**价值**：
- AI 不会重复犯同样的错
- 自动执行，无需提醒
- 保证系统稳定性

**命名规范**：`强制规则_[领域].md`

**示例**：
```
强制规则/
├── 强制规则_智能存储路由.md
├── 强制规则_任务与想法追踪.md
└── 强制规则_创作任务触发条件.md
```

---

## 🔄 记忆提炼流程

```
情景记忆 → 识别模式 → 语义记忆
语义记忆 → 发现必须遵守的规则 → 强制规则
```

### 触发时机
- 每次自我迭代时
- 发现重复模式时（≥3次）
- 完成重大项目后

## 📊 记忆资产的价值

### ❌ 没有这个结构
记忆就是一堆文件，扔进去就找不到了

### ✅ 有了这个结构
记忆变成：
- **可检索**：按类型、时间、主题快速找到
- **可复用**：提炼的知识可以在不同场景应用
- **可进化**：通过迭代不断优化和更新

## 💡 使用示例

### 场景：你和 AI 讨论了 "如何写爆款标题"

#### ❌ 第一代的做法
```
记忆库/
└── 2026-02-13-爆款标题讨论.md
```
扔进去，以后大概率找不到。

#### ✅ 第二代的做法
```
记忆库/
├── 情景记忆/2026-02/
│   └── 0213-讨论爆款标题.md          # 完整对话过程
├── 语义记忆/
│   └── 爆款标题方法论.md              # 提炼的通用方法
└── 强制规则/
    └── 强制规则_标题创作流程.md       # 必须遵守的步骤
```

**价值**：
- 情景记忆让你能回溯当时怎么想的
- 语义记忆让知识可以复用，不用每次重新讨论
- 强制规则让 AI 不会重复犯同样的错

---

## 🗂️ 归档规则

### 情景记忆
- 按月归档：`情景记忆/YYYY-MM/`
- 每月初创建新月度目录
- 原始文件保留，不删除

### 语义记忆
- 不按时间归档
- 按主题分类
- 定期更新和优化

### 强制规则
- 永久保存
- 按领域分类
- 系统自动执行

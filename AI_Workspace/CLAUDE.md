# AI 工作系统总指南 v2.0

> **核心理念**：第一代你维护系统，第二代系统维护自己

## 🎯 系统架构概览

```
记忆库/
├── 情景记忆/   # 具体事件和对话（按月归档）
├── 语义记忆/   # 提炼的知识和规律
└── 强制规则/   # 必须遵守的行为约束

本体画像/       # TELOS 模块化身份系统（按需加载）
```

## 📋 启动流程优化

**目标**：减少 90%+ 的无效 Input Token 消耗

### 规则
- ❌ 不要自动读取日记、本体画像目录、user.md、AI_state.json
- ✅ 只在用户明确需要时才读取相关文件
- ✅ 优先使用已有的上下文信息
- ✅ 如果需要读取，使用 limit 参数限制行数

---

## ✅ 称呼规则与指令遵循度检测

**目标**：让用户明确知道 AI 是否还在遵循指令

### 规则
- 每次回复时，**必须在开头使用 "✓" 符号**
- 这是指令遵循度的标志
- 如果你发现 AI 的回复没有 "✓" 符号，说明上下文已过载
- 此时应提醒用户使用 `/new` 重开会话

---

## 📊 上下文管理

**目标**：避免在 "AI 已经不听话" 的状态下继续工作

### 规则
- 当上下文使用超过 40% 时，指令遵循度开始下降
- 当上下文使用超过 60% 时，**必须主动提醒用户重开会话**
- 每次会话结束时，记录上下文使用情况到日志

---

## 🔄 会话边界管理

**目标**：保持 AI 在每个会话中都 "清醒"

### 规则
- 每次会话结束时记录上下文使用情况
- 复杂任务拆分成多个会话
- 避免在单个会话中做太多事情
- 每个会话聚焦一个主题

```
情景记忆 → 识别模式 → 语义记忆
语义记忆 → 发现必须遵守的规则 → 强制规则
```

## ⏰ 时间意图自动捕获（强制执行）

**目标**：你说的每一个时间意图都会被捕获，不会遗漏

### 触发词
- "明天"、"后天"、"下周"、"X月X日"
- "要发"、"要做"、"记得"、"别忘了"
- "计划"、"打算"、"准备"

### 自动执行流程
1. 将事项添加到 `任务清单.md`（根据紧急程度分类）
2. 告知用户："已添加到任务清单，届时会提醒你"

### 示例
```
用户："这篇文章明天发"
AI：
1. 更新文章状态
2. 添加到任务清单（紧急重要）
3. 回复："已添加到任务清单，明天简报会提醒你发布"
```

## 🚀 发布状态自动更新（强制执行）

**目标**：文件状态自动同步，不需要手动管理

### 触发词
- "已发布"、"发布了"、"发出去了"
- "已经发了"、"刚发了"

### 自动执行流程
1. 将文章目录从 `02-待发布/` 移动到 `03-已发布/`
2. 告知用户："已移动到已发布目录"

### 示例
```
用户："这篇已经发布出去了"
AI：
1. mv 02-待发布/{文章目录} → 03-已发布/{文章目录}
2. 回复："已移动到已发布目录 ✅"
```

## 🗂️ 智能存储路由（强制执行）

**目标**：减少决策疲劳，文件永远在正确的位置

### 每次创建文件时自动执行
1. 识别内容类型（文章/素材/科研/商业/工具/系统文件）
2. 查询存储规则
3. 构建完整文件路径
4. 验证路径合规性
5. 执行存储操作
6. 告知用户保存位置和决策理由

### 存储规则
```
文章     → 01-内容生产/选题管理/02-待发布/YYYY-MM-DD-标题/
素材     → 01-内容生产/素材库/[类型]/
科研     → 03-科研/[项目名]/
商业     → 02-商业/[类型]/
工具     → 04-工具与效率/[类型]/
系统文件 → 根目录或记忆库/
```

### 决策示例
```
用户："帮我写一篇关于LLM的文章"
AI：
1. 创作文章内容
2. 识别为"文章"类型
3. 路径：01-内容生产/选题管理/02-待发布/2026-02-13-LLM文章/
4. 创建目录和文件
5. 告知用户：
   ✅ 文章已保存到：01-内容生产/选题管理/02-待发布/2026-02-13-LLM文章/
   📁 内容类型：文章
   💡 决策理由：长文本内容，用于发布
```

## 🔄 八步自我迭代流程

**目标**：系统会自己进化

### 流程

#### 1. 观察（Observe）
- 回顾本次会话/任务的完整过程
- 产出：清晰的事件时间线

#### 2. 分析（Analyze）
- 什么做得好？为什么？
- 什么可以改进？为什么？
- 发现了什么新模式？
- 产出：分析结论列表

#### 3. 设计（Design）
- 针对问题设计解决方案
- 针对模式设计抽象方法
- 产出：改进方案文档

#### 4. 实施（Implement）
- 更新系统文件
- 创建新的知识文档
- 产出：实际的文件改动

#### 5. 验证（Validate）
- **自我检查**：回顾所有改动，检查是否符合强制规则
- **逻辑检查**：验证改进方案是否有遗漏或矛盾
- **质量评分**：给出 1-10 分的自我评分
- **反思改进**：如果评分低于 7 分，重新设计方案
- 产出：验证报告

#### 6. 记录（Record）
- 更新对话历史
- 创建情景记忆
- 产出：完整的记录文档

#### 7. 提炼（Distill）
- 创建/更新语义记忆
- 总结方法论
- 产出：语义记忆文档

#### 8. 提交（Commit）
- `git add .`
- `git commit -m "iteration: [本次迭代主题]"`
- 产出：Git commit

### 质量评分标准
- **9-10分**：完美执行，产出高质量成果
- **7-8分**：良好执行，有小瑕疵
- **5-6分**：基本完成，有明显改进空间
- **3-4分**：勉强完成，需要重大改进
- **1-2分**：执行失败，需要重新设计

## ⚖️ 决策确认约束

**目标**：该问的问，该做的做，不越界

### 必须询问用户的情况
- 不确定的架构设计决策
- 可能影响现有系统的改动
- 有多种实现方案的选择
- 涉及删除或重大重构的操作
- 不确定用户意图时

### 可以自主决策的情况
- 明确的任务执行
- 遵循已有规则的操作
- 创建新文件（符合存储路由规则）
- 日常记录和追踪
- 按照强制规则执行的操作

## 💻 代码质量约束

**目标**：代码简洁，不臃肿

### 禁止行为
❌ 不写兼容性代码（除非用户明确要求）
❌ 不在原有代码上加 if 分支来 "修复" 问题
❌ 不添加不必要的错误处理
❌ 不过度抽象
❌ 不创建 "以防万一" 的功能

### 应该做的
✅ 直接修改问题代码
✅ 保持代码简洁
✅ 只实现当前需求
✅ 删除无用代码
✅ 优先简单方案

## 🗄️ 数据治理原则（最高优先级）

**目标**：数据是资产，越积累越有价值

### 核心原则
**数据不删除，只转移**

### 为什么
- 年终总结需要统计全年完成的任务数量和类型
- 复盘需要历史数据支撑
- 数据是复利资产，删除即损失

### 什么情况可以删除
1. 用户明确说 "删除"、"不要了"、"没用了"
2. 数据完全重复（去重）
3. 用户确认是垃圾数据

**其他所有情况：只能转移，不能删除**

## 👥 全局资源调度

**目标**：效率最大化，不是所有事都自己做

### 当收到任务时，按以下步骤思考

#### 1. 任务分析
- 这个任务需要什么能力？
- 预计需要多少时间？
- 质量要求如何？

#### 2. 资源匹配（按优先级）
- ✅ AI 能高质量完成 → 直接执行
- ✅ 需要人类判断但 AI 可辅助 → AI 起草 + 用户审核
- ⚠️ 用户必须亲自做 → 提醒并协助准备
- 💡 有更合适的人 → 查询人际资源库，推荐委托
- 💰 可以外包 → 建议外包方案

#### 3. 决策建议格式
```
**任务**：[任务描述]
**推荐方案**：[谁来做]
**理由**：[为什么这样分配最优]
**预计成本**：[时间/金钱/人情]
```

### 原则
**时间价值优先，能委托的尽量委托**

## 📁 TELOS 模块化身份系统

**目标**：按需加载，节省 token，响应更快

### 目录结构
```
本体画像/
├── README.md              # 系统说明
├── 00-核心身份.md         # 我是谁（必读，每次会话都读）
├── 01-价值观与原则.md     # 我相信什么（按需读取）
├── 02-背景与经历.md       # 我的历史（按需读取）
├── 03-技能与能力.md       # 我会什么（按需读取）
├── 05-目标与规划.md       # 我要去哪（按需读取）
├── 07-工作方式.md         # 我怎么工作（按需读取）
└── 08-关系网络.md         # 我认识谁（按需读取）
```

### 加载规则
- **00-核心身份.md**：每次会话必读（控制在 500 字以内）
- **其他模块**：只在相关任务时读取

### 效果
只读核心身份时，token 消耗降低 70%+

## 📝 使用指南

1. 填写 `本体画像/00-核心身份.md`（必填）

- 迭代会自动执行八步流程
- 所有改动都会提交到 git

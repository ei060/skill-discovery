## 🎯 为什么模块化？

### ❌ 第一代：单文件模式
```markdown
个人定位.md
（所有信息塞在一个文件里，AI 每次都要读完）
```

**问题**：
- AI 每次会话都要读取全部信息
- 浪费大量 token（99% 消耗在 Input 上）
- 更新困难，改动影响整个文件

### ✅ 第二代：TELOS 模块化
```
本体画像/
├── 00-核心身份.md         # 必读
├── 01-价值观与原则.md     # 按需
├── 02-背景与经历.md       # 按需
└── ...
```

**优势**：
- AI 只读需要的模块
- 节省 token，响应更快
- 可以单独更新某一部分

## 📋 加载规则

### 必读模块
**00-核心身份.md**
- 每次会话必读
- 控制在 500 字以内
- 定义最核心的身份信息

### 按需模块
**01-08 其他模块**
- 只在相关任务时读取
- 不需要就不读取
- 大幅降低 token 消耗

**效果**：只读核心身份时，token 消耗降低 70%+

## 🎭 TELOS 模型说明

TELOS 是古希腊语中的 "目的" 或 "目标"，这里用作身份系统的框架：

### T - Trait（特质）
我是谁？我的核心特质是什么？
→ **00-核心身份.md**

### E - Ethics（伦理）
我相信什么？我的价值观和原则是什么？
→ **01-价值观与原则.md**

### L - Life（生命经历）
我经历过什么？什么塑造了现在的我？
→ **02-背景与经历.md**

### O - Objectives（目标）
我要去哪里？我的短期和长期目标是什么？
→ **05-目标与规划.md**

### S - Skills（技能）
我会什么？我的核心能力和技能是什么？
→ **03-技能与能力.md**

### 补充模块
**07-工作方式.md**：我怎么工作？我的工作风格和偏好是什么？
**08-关系网络.md**：我认识谁？我的人际网络和资源有哪些？

## 💡 使用指南

### 首次设置
1. **必填**：填写 `00-核心身份.md`（控制在 500 字以内）
2. **推荐**：填写 `01-价值观与原则.md` 和 `05-目标与规划.md`
3. **可选**：根据需要填写其他模块

### 日常使用
- AI 会自动在会话开始时读取 `00-核心身份.md`
- 当任务涉及特定领域时，AI 会自动读取相关模块
- 不需要手动告诉 AI 读取哪个模块

### 更新维护
- 可以随时更新任何模块
- 更新后 AI 会自动使用最新信息
- 建议定期回顾和更新（如每季度）

## 🔗 与其他系统的关系

```
本体画像 → 定义 "我是谁"
    ↓
记忆库 → 记录 "我学到了什么"
    ↓
任务系统 → 追踪 "我要做什么"
```

- **本体画像** 是静态的自我认知
- **记忆库** 是动态的知识积累
- **任务系统** 是行动的执行追踪

## ⚠️ 注意事项

1. **简洁优先**：每个模块都要简洁明了，避免冗长
2. **定期更新**：随着自我认知的加深，定期更新内容
3. **真实坦诚**：诚实地面对自己，不要虚构或美化
4. **聚焦核心**：只记录最重要的信息，细节可以放在记忆库

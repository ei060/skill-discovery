## 🎯 系统简介

这是一套基于 Claude Code 的第二代 AI 工作系统，专为**内容创作者**设计。

### 核心特性

✅ **自我进化**：AI 自动发现 → 分析 → 修复 → 验证
✅ **懒加载**：减少 90%+ 的无效 token 消耗
✅ **智能路由**：文件自动分类，永远在正确位置
✅ **时间捕获**：每个承诺都会被记住
✅ **数据资产**：只转移不删除，积累复利

## 🎓 核心概念

### 三层记忆架构

```
情景记忆 → 语义记忆 → 强制规则
(事件)    (知识)     (规则)
```

- **情景记忆**：具体事件的完整记录
- **语义记忆**：提炼的可复用知识
- **强制规则**：必须遵守的行为约束

### TELOS 模块化

按需加载，节省 token：
- **00-核心身份.md**：必读，每次会话都读
- **其他模块**：只在相关任务时读取

### 八步自我迭代

1. **观察**：回顾完整过程
2. **分析**：发现问题和模式
3. **设计**：设计解决方案
4. **实施**：更新系统文件
5. **验证**：自我检查和评分
6. **记录**：创建情景记忆
7. **提炼**：创建语义记忆
8. **提交**：Git commit

## 🛠️ 技术细节

### 指令遵循度检测

每次回复开头都有 **✓** 符号，这是 AI 是否还在遵循指令的标志。

如果没有 ✓，说明上下文已过载，应该重开会话。

### 上下文管理

- **40%**：指令遵循度开始下降
- **60%**：必须提醒用户重开会话
- **80%**：AI 可能开始 "不听话"

### 数据治理原则

**数据不删除，只转移**

只有以下情况可以删除：
1. 用户明确说 "删除"
2. 数据完全重复（去重）
3. 用户确认是垃圾数据

## 两个核心组件

### 1. openn - MCP Server (后端数据源)

**仓库**: github.com/6551Team/openn
**Skill**: clawhub.ai/infra403/openn
**作者**: @infra403

**功能**:
- ✅ X (Twitter) 实时数据接入
- ✅ 全网 50+ 新闻源聚合
- ✅ 链上数据 (On-chain data)
- ✅ 24h 监控和分析
- ✅ Telegram 提醒触发
- ✅ **无需 API 密钥配置**

**技术特点**:
- MCP (Model Context Protocol) 服务器
- 数据基础架构积累 1 年
- 开箱即用

### 2. opent - OpenClaw Skill (前端接口)

**仓库**: github.com/6551Team/opent
**Skill**: clawhub.ai/infra403/opent
**作者**: @infra403

**功能**:
- OpenClaw 前端 Skill
- 连接 MCP 服务器
- 自动新闻推送
- 智能内容过滤
- 几分钟部署完成

## 解决的问题

### 问题 1: X API 接入难 ❌ → ✅
**传统方案**:
- 需要申请 API 密钥
- 配置复杂
- 有调用限制
- 费用高昂

**6551 解决**:
- 无需 API 密钥
- 开箱即用
- 团队统一维护

### 问题 2: Skill 开发门槛高 ❌ → ✅
**传统方案**:
- 需要编程知识
- 调试复杂
- 维护困难

**6551 解决**:
- 开箱即用 Skill
- 几分钟部署
- 一行命令安装

### 问题 3: 信息过载 ❌ → ✅
**传统方案**:
- 新闻太多看不完
- 手动筛选耗时
- 容易错过重要信息

**6551 解决**:
- AI 自动过滤
- 24h 监控
- TG 智能提醒

### 问题 4: 24/7 监控需求 ❌ → ✅
**传统方案**:
- 需要持续运行脚本
- 服务器成本高
- 维护困难

**6551 解决**:
- MCP 服务器持续运行
- 统一数据处理
- 多渠道推送

## 🔍 ECC Hooks 去哪了？

### 重要发现

ECC (everything-claude-code) 的 hooks **不存储在** settings.local.json 中。

ECC 使用两种方式加载 hooks：

1. **SessionStart Hook**: 动态添加 hooks 到配置
   ```json
   "SessionStart": [
     {
       "matcher": "*",
       "hooks": [
         {
           "type": "command",
           "command": "bash -lc '...run-with-flags.js...'"
         }
       ]
     }
   ]
   ```

2. **run-with-flags.js**: 运行时动态调用 ECC 插件中的 hook 脚本
   - 路径: `~/.claude/plugins/marketplaces/everything-claude-code/scripts/hooks/`
   - 例如: `pre-bash-tmux-reminder.js`, `post-bash-pr-created.js`

### 结论

- ECC hooks 不需要静态配置在 settings.local.json
- AI Roland hooks 需要（因为我们不是插件）
- 当前配置是正确的：AI Roland hooks 在 settings.local.json，ECC hooks 通过插件系统加载

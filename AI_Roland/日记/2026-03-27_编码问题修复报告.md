## 一、问题诊断

### 1.1 问题现象

**用户报告**：
- 记忆文件中的中文显示为乱码
- 示例：`"鎺㈢储"` 应该是 `"列出"`

**影响范围**：
- 18 个 Agent 记忆文件
- 总计 68 条记忆记录
- 其中 5 条严重损坏

### 1.2 根因分析

**问题代码**：
```python
# ❌ 错误代码（save_memory.py 和 inject_memory.py）
def read_stdin_json():
    stdin_data = sys.stdin.read()  # 使用系统默认编码（Windows GBK）
    return json.loads(stdin_data)
```

**问题流程**：
```
1. Claude Code 发送 UTF-8 编码的 JSON
2. Python sys.stdin.read() 使用系统默认编码（GBK）解码
3. UTF-8 字节被错误解析为 GBK 字符
4. 乱码数据保存到记忆文件
```

**示例**：
```
原文: "探索" (UTF-8: e68ea2e7b4a2)
      ↓ 被 GBK 解码
乱码: "鎺㈢储" (UTF-8: e98ebae388a2e582a8)
```

## 二、修复方案

### 2.1 代码修复

**修复代码**：
```python
# ✅ 正确代码
def read_stdin_json():
    """使用 sys.stdin.buffer 读取原始字节，然后 UTF-8 解码"""
    try:
        raw_bytes = sys.stdin.buffer.read()  # 读取原始字节
        if not raw_bytes:
            return None
        stdin_data = raw_bytes.decode('utf-8')  # 显式使用 UTF-8
        return json.loads(stdin_data)
    except Exception as e:
        return None
```

## 六、技术总结

### 6.1 关键学习

**Windows 编码问题**：
- 系统默认编码：CP936 (GBK)
- Python sys.stdin：继承系统默认编码
- 正确做法：使用 sys.stdin.buffer + 显式 UTF-8 解码

**修复模式**：
```python
# ❌ 错误
data = sys.stdin.read()  # 依赖系统编码

# ✅ 正确
raw = sys.stdin.buffer.read()  # 原始字节
data = raw.decode('utf-8')     # 显式编码
```

### 6.2 防范措施

**最佳实践**：
1. 所有 I/O 操作显式指定编码
2. 使用原始字节模式再手动解码
3. 添加编码验证测试
4. 定期检查数据质量

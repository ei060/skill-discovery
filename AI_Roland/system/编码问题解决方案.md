# Windows 控制台中文乱码解决方案

## 问题原因

Windows 命令提示符（cmd）默认使用 **GBK** 编码，而 Python 脚本输出的是 **UTF-8** 编码，导致中文显示为乱码。

## 解决方案

### 方案 1：使用 UTF-8 包装脚本（推荐）

```bash
# 使用 run_utf8.bat 运行任何 Python 脚本
run_utf8.bat skills_manager.py
run_utf8.bat hooks_manager.py
run_utf8.bat memory_search.py
```

### 方案 2：设置环境变量

```bash
# 临时设置
set PYTHONIOENCODING=utf-8
python skills_manager.py

# 或永久设置（添加到系统环境变量）
PYTHONIOENCODING=utf-8
```

### 方案 3：使用 PowerShell

PowerShell 默认支持 UTF-8：

```powershell
cd AI_Roland\system
python skills_manager.py
```

### 方案 4：在代码中修复（已实现）

所有系统脚本已添加编码修复代码：

```python
import sys
import os
if sys.platform == 'win32':
    os.environ['PYTHONIOENCODING'] = 'utf-8'
    import io
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8')
```

## 推荐使用方式

### 1. 创建快捷脚本

创建 `test_skills.bat`：

### 2. 使用 PowerShell

创建 `test_skills.ps1`：

### 3. 直接在 Python 中测试

```python

manager = SkillsManager()
skills = manager.list_skills()

# 输出到文件（避免控制台编码问题）
with open('skills_output.txt', 'w', encoding='utf-8') as f:

        f.write(f"- {skill['name']}: {skill['description']}\n")

print("结果已保存到 skills_output.txt")
```

## 常见问题

**Q: 为什么在 PowerShell 中正常，在 cmd 中乱码？**

A: PowerShell 默认使用 UTF-8，cmd 默认使用 GBK。使用 PowerShell 或设置 `chcp 65001`。

**Q: 设置后还是乱码？**

A: 确保在设置编码后**重启**命令行窗口。

**Q: 日志文件中文乱码？**

A: 确保日志文件以 UTF-8 编码保存：
```python
with open(log_file, 'w', encoding='utf-8') as f:
    f.write("中文内容")
```

## 完美解决方案

**最佳实践**：使用 Python 包管理工具运行：

```bash
# 在项目根目录创建 pyproject.toml
```

这样可以避免所有编码问题。

# AI Roland 自动启动机制

## 概述

每次启动 Claude 时，AI Roland 系统会自动：
1. ✅ 检查并启动守护进程（如未运行）
2. ✅ 加载最近对话历史摘要
3. ✅ 显示当前任务状态
4. ✅ 生成今日简报
5. ✅ 记录新会话开始

## 使用方法

### 方法 1：手动启动脚本（推荐）

在每次 Claude 会话开始时，运行以下命令：

```bash
python AI_Roland/system/session_start.py
```

或者双击运行：
```
启动AI_Roland.bat
```

### 方法 2：在 Claude 中导入

在 Claude 会话开始时，使用 Python 代码：

```python
import sys
sys.path.insert(0, 'AI_Roland/system')
from session_start import SessionStarter

starter = SessionStarter()
result = starter.start_session()
print(result["briefing"])
```

## 自动记录会话

每次运行启动脚本时，会自动在 `对话历史.md` 中追加新会话记录：

```markdown
---
### 会话：2026-02-20 16:54
**用户**：会话开始
**AI Roland**：系统已启动，等待用户输入...
**任务**：待记录
**产出**：待更新
---
```

## 守护进程自动管理

- **如果守护进程未运行**：自动启动
- **如果守护进程已运行**：跳过启动
- **进程状态检查**：通过 PID 验证进程是否存活

## 文件结构

```
AI_Roland/
├── system/
│   ├── session_start.py      # 会话启动脚本
│   ├── daemon.py             # 守护进程
│   ├── task_utils.py         # 任务统计工具
│   └── 自动启动说明.md       # 本文档
├── 对话历史.md               # 自动追加会话记录
├── 任务清单.md               # 自动更新统计
├── daemon_status.json        # 守护进程状态
└── system_state.json         # 系统状态
```

## 优点

1. **零配置**：无需手动启动守护进程
2. **上下文恢复**：自动显示最近对话和任务
3. **状态监控**：实时了解系统运行状态
4. **会话记录**：自动保存对话历史
5. **任务同步**：自动更新任务统计

## 注意事项

1. 确保 Python 环境已安装
2. 首次运行需要启动守护进程
3. 会话记录会持续追加，不会覆盖历史
4. 守护进程在后台持续运行，不受会话影响

## 故障排查

### 守护进程无法启动
- 检查 Python 路径是否正确
- 查看日志文件：`AI_Roland/logs/daemon_YYYYMMDD.log`
- 手动测试：`python AI_Roland/system/daemon.py`

### 会话记录未追加
- 检查文件权限
- 确保 `对话历史.md` 文件存在
- 查看脚本输出错误信息

### 任务统计不准确
- 手动运行：`python AI_Roland/system/task_utils.py`
- 检查任务清单格式是否正确
- 守护进程会在下次心跳时自动更新

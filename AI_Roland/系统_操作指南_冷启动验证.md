# 冷启动验证测试指南

**用途**: 验证 Claude Code 重启后 Hook 链自动恢复
**预计耗时**: 5 分钟
**难度**: ⭐ 简单

---

## 📋 测试前准备

### 确认当前状态

在重启前，先记录当前状态：

```bash
# 1. 检查当前日志文件的最后修改时间
ls -la AI_Roland/system/agents/hooks/*.log

# 2. 记录 current_task.json 的内容
cat AI_Roland/current_task.json

# 3. 记录 SESSION_CONTEXT.md 的修改时间
ls -la .claude/SESSION_CONTEXT.md
```

**保存这些信息**，以便重启后对比。

---

## 🔄 步骤 1: 重启 Claude Code

### Windows

1. **完全关闭 Claude Code**
   - 关闭所有 Claude Code 窗口
   - 检查任务管理器，确保没有残留进程

2. **重新启动 Claude Code**
   - 从开始菜单启动
   - 或双击桌面图标

3. **打开工作空间**
   - 选择 `D:/ClaudeWork` 目录
   - 等待加载完成

## ❓ 故障排查

### 问题 1: SESSION_CONTEXT.md 未生成

**可能原因**:
- restore_session.py 未执行
- 配置文件路径错误

**解决方法**:
```bash
# 检查配置文件
cat .claude/settings.local.json | grep restore_session

# 检查文件是否存在
ls -la AI_Roland/system/agents/hooks/restore_session.py

# 手动测试
echo '{}' | python AI_Roland/system/agents/hooks/restore_session.py
```

### 问题 2: task_dispatcher 未捕获 TaskCreate

**可能原因**:
- task_dispatcher.py 未执行
- Python 路径问题

**解决方法**:
```bash
# 检查 Python 路径
where python

# 手动测试
echo '{"tool_name":"TaskCreate"}' | python AI_Roland/system/task_dispatcher.py

# 检查错误日志
tail -20 AI_Roland/system/agents/hooks/dispatcher_errors.log
```

### 问题 3: 记忆注入未触发

**可能原因**:
- inject_memory.py 的 matcher 配置错误
- Task tool 未被识别

**解决方法**:
```bash
# 检查配置
cat .claude/settings.local.json | grep -A 5 "matcher.*Task"

# 确认使用的是 Task tool（不是 TaskCreate）
# 正确: Task(subagent_type="Explore", ...)
# 错误: TaskCreate(...)
```

### 问题 4: 所有 hooks 都未执行

**可能原因**:
- 配置文件格式错误
- 配置文件未加载

**解决方法**:
```bash
# 验证 JSON 格式
python -m json.tool .claude/settings.local.json

# 检查文件权限
ls -la .claude/settings.local.json

# 查看完整配置
cat .claude/settings.local.json
```

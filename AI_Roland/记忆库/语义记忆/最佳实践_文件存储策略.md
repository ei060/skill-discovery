# 文件存储策略

## 核心原则

**所有文件必须保存在 AI_Roland/ 目录内**

这是宪法级别的规则，任何情况下不得违反。

## 存储路由系统

### 自动识别类型

创建文件时，系统自动识别内容类型：

| 类型关键词 | 路由规则 |
|-----------|---------|
| 文章、博客、发布 | 01-内容生产/选题管理/02-待发布/ |
| 素材、图片、资源 | 01-内容生产/素材库/[类型]/ |
| 科研、研究、论文 | 03-科研/[项目名]/ |
| 商业、产品、运营 | 02-商业/[类型]/ |
| 工具、脚本、效率 | 04-工具与效率/[类型]/ |
| 系统文件、配置 | 根目录或 system/ |
| 记忆、知识 | 记忆库/[类型]/ |

### 存储决策流程

```
用户请求创建文件
    ↓
识别内容类型
    ↓
查询存储规则
    ↓
构建完整路径
    ↓
验证路径合规性
    ↓
执行存储操作
    ↓
告知用户位置和理由
```

## 实践案例

### 案例1：创作文章

**用户请求**：帮我写一篇关于LLM的文章

**系统处理**：
1. 创作文章内容
2. 识别为"文章"类型
3. 路径：01-内容生产/选题管理/02-待发布/2026-02-20-LLM文章/
4. 创建目录和文件
5. 回复用户：
```
✅ 文章已保存到：01-内容生产/选题管理/02-待发布/2026-02-20-LLM文章/
📁 内容类型：文章
💡 决策理由：长文本内容，用于发布
```

### 案例2：保存工具脚本

**用户请求**：帮我写一个批量重命名脚本

**系统处理**：
1. 编写脚本代码
2. 识别为"工具"类型
3. 路径：04-工具与效率/脚本工具/
4. 创建文件
5. 回复用户：
```
✅ 脚本已保存到：04-工具与效率/脚本工具/batch_rename.py
📁 内容类型：工具脚本
💡 决策理由：可复用的自动化脚本
```

### 案例3：记录科研笔记

**用户请求**：记录刚才关于多智能体的想法

**系统处理**：
1. 整理笔记内容
2. 识别为"科研"类型
3. 路径：03-科研/多智能体研究/
4. 创建文件

```
💡 决策理由：研究相关的思考和想法
```

## 边界规则

### 禁止行为（绝对）

❌ 将文件保存到 AI_Roland 目录外
❌ 保存到 ~/Dev/ 下的任何路径
❌ 在根目录直接保存文件
❌ 不告知用户文件保存位置

### 唯一例外

**只读操作**：读取外部文件用于分析是允许的

## 发布状态管理

### 待发布 → 已发布

**触发词**：
- "已发布"、"发布了"、"发出去了"
- "已经发了"、"刚发了"

**自动执行**：
1. 将文章目录从 02-待发布/ 移动到 03-已发布/
2. 更新文件结构（如有必要）
3. 告知用户

**示例**：
```
用户："这篇文章已经发布了"
AI：
1. mv 02-待发布/2026-02-20-LLM文章/ → 03-已发布/2026-02-20-LLM文章/
2. 回复："已移动到已发布目录 ✅"
```

## 错误处理

### 用户指定外部路径

**错误示例**：
```
用户："把文件保存到 C:\Users\Desktop\file.txt"
```

**正确响应**：
```
❌ 这违反了 AI Roland 的工作区边界规则，我无法执行。
✅ 建议保存到：AI_Roland/04-工具与效率/文件名.txt
💡 这样可以保持所有文件在统一管理范围内。
```

### 用户坚持外部路径

**响应**：
```
我理解你的需求，但这违反了系统的核心规则。
工作区边界是为了：
1. 统一管理所有文件
2. 避免文件散落丢失
3. 便于备份和迁移
4. 保持系统完整性

如果你确实需要外部文件，可以手动复制，但我无法直接保存到外部路径。
```

## 每周复盘机制

**执行时间**：每周一晨间简报时

**检查内容**：
1. 扫描 AI_Roland/ 目录
2. 识别错位文件
3. 统计错位数量
4. 生成复盘报告

**报告格式**：
```markdown
# 每周存储复盘
日期：2026-02-20

## 错位文件统计
- 总数：X 个
- 类型分布：
  - 文章错位：Y 个
  - 工具错位：Z 个

## 错位文件列表
1. [路径] - 应该移到 [正确路径]

## 改进计划
- [ ] 移动错位文件
- [ ] 更新存储规则
- [ ] 优化自动识别
```

## 注意事项

⚠️ **路径合规性验证**：创建文件前必须验证路径
⚠️ **决策透明化**：告知用户为什么选择这个路径
⚠️ **定期复盘**：每周检查并优化
⚠️ **规则更新**：发现新模式时更新规则

## 改进概述

## 问题背景

### 原有问题

在 v2.0 中，搜索结果仅基于语义相似度排序，不考虑文档的时间新鲜度。

**问题场景**：
- 用户搜索"守护进程"
- 系统返回了3个月前的旧配置文档
- 但昨天刚更新了新配置，却排在后面

**用户期望**：
- 最近的记忆应该更相关
- 新内容应该优先展示
- 历史内容可以保留但不应优先

## 解决方案

### 时间权重算法

```python
def calculate_time_weight(self, doc_date: str, half_life_days: int = 30) -> float:
    """计算时间权重

    Args:
        doc_date: 文档日期（YYYY-MM-DD 格式）
        half_life_days: 半衰期天数，默认30天

    Returns:
        时间权重系数（0-1之间，越近的文档权重越高）
    """
    if not doc_date:
        return 0.5  # 没有日期的文档使用中等权重

    try:
        doc_datetime = datetime.strptime(doc_date, "%Y-%m-%d")
        days_ago = (datetime.now() - doc_datetime).days
        # 指数衰减公式
        time_weight = 1 / (1 + days_ago / half_life_days)
        return time_weight
    except (ValueError, TypeError):
        return 0.5
```

### 权重公式

**综合相似度 = 语义相似度 × (0.7 + 0.3 × 时间权重)**

**设计理由**：
- 基础权重 0.7：保证语义相关性仍然是主要因素
- 时间权重 0.3：时间新鲜度作为次要因素
- 避免时间权重过大导致不相关的新内容排在前面

### 时间权重曲线

| 天数前 | 时间权重 | 综合系数 | 说明 |
|--------|---------|---------|------|
| 0（今天） | 1.0 | 1.0 | 最高权重 |
| 7天前 | 0.81 | 0.94 | 本周内容 |
| 30天前 | 0.5 | 0.85 | 一个月前 |
| 60天前 | 0.33 | 0.80 | 两个月前 |
| 90天前 | 0.25 | 0.78 | 三个月前 |
| 180天前 | 0.14 | 0.74 | 半年前 |
| 365天前 | 0.076 | 0.72 | 一年前 |

**特点**：
- 前30天：权重快速下降
- 30天后：下降速度变缓
- 不会完全排除旧内容，只是降低优先级

## 技术细节

### 日期解析

**支持的日期格式**：`YYYY-MM-DD`

**示例**：
- `2026-02-20` ✅
- `2026-02` ❌（不支持）
- `02/20/2026` ❌（不支持）

**异常处理**：
- 日期解析失败 → 返回 0.5（中等权重）
- 没有日期字段 → 返回 0.5（中等权重）

### 半衰期参数

**默认值**：30 天

**含义**：30天前的文档时间权重为 0.5

**可调整**：
```python
# 更短的半衰期（更快衰减）
time_weight = self.calculate_time_weight(doc_date, half_life_days=14)

# 更长的半衰期（更慢衰减）
time_weight = self.calculate_time_weight(doc_date, half_life_days=60)
```

### API 变化

**v2.0**：
```python
results = search.search("任务 管理", top_k=5)
```

**v2.1**（向后兼容）：
```python
# 默认使用时间权重
results = search.search("任务 管理", top_k=5)

# 禁用时间权重
results = search.search("任务 管理", top_k=5, use_time_weight=False)
```

## 使用场景

### 适用场景

✅ **推荐使用时间权重**：
1. **快速查找最近的内容**：
   - "我昨天写的关于 Hooks 的笔记在哪？"
   - "最近有没有讨论过守护进程？"

2. **优先展示新知识**：
   - 用户更可能需要最新的信息
   - 避免使用过时的配置或方法

3. **持续更新的系统**：
   - 知识库频繁更新
   - 新内容比旧内容更重要

### 不适用场景

❌ **不建议使用时间权重**：
1. **查找历史记录**：
   - "上个月是怎么配置的？"
   - 查看具体某天的讨论

2. **经典知识查询**：
   - 基础概念、方法论
   - 不随时间变化的知识

**解决方案**：使用 `use_time_weight=False`

## 已知限制

### 当前限制

1. **日期格式固定**：
   - 只支持 `YYYY-MM-DD` 格式
   - 不支持其他日期格式

2. **没有日期的文档**：
   - 统一使用中等权重（0.5）
   - 可能不准确

3. **语义记忆没有日期**：
   - 语义记忆文件通常没有日期字段
   - 无法享受时间权重优势

### 未来改进

**可选改进**：
1. 支持更多日期格式（ISO 8601、时间戳等）
2. 从文件名提取日期（如 `2026-02-20-笔记.md`）
3. 从文件修改时间提取日期
4. 添加时间衰减曲线配置（线性、指数、对数）

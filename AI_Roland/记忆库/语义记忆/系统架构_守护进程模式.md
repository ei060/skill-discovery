# 守护进程模式

## 概念

守护进程是在后台持续运行的系统组件，负责周期性任务和系统维护。

## 核心特征

1. **独立性**：与主进程分离，独立运行
2. **周期性**：按固定间隔执行任务（如每60秒心跳）
3. **持久化**：进程状态保存在 JSON 文件中
4. **可恢复**：支持启动、停止、重启操作

## 实现要点

### 进程管理
- 使用 PID 文件跟踪进程状态
- 支持优雅关闭（SIGTERM）
- 防止重复启动

### 心跳机制
- 定期更新心跳计数和最后活动时间
- 用于监控进程健康状态
- 可用于检测进程异常

### 状态文件
```json
{
  "pid": 12345,
  "last_heartbeat": "2026-02-20T10:30:00",
  "heartbeat_count": 42,
  "status": "running"
}
```

### 集成功能
- 任务统计自动更新
- 数据维护提醒
- 系统健康检查

## 使用场景

1. **自动统计**：周期性更新任务清单的统计数字
2. **定时任务**：每日简报、周日数据维护提醒
3. **状态监控**：守护系统运行状态
4. **数据同步**：定期同步或备份数据

## 注意事项

- ⚠️ Windows 下需要使用 pythonw.exe 避免弹出窗口
- ⚠️ 确保只有唯一实例运行
- ⚠️ 处理异常时保持进程稳定
- ⚠️ 日志记录便于排查问题

## 相关文件

- `daemon.py` - 守护进程实现
- `daemon_status.json` - 状态文件
- `start_daemon.bat` - 启动脚本
- `install_service.py` - Windows 服务安装

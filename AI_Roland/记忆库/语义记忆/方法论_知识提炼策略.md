# 知识提炼策略

## 概念

知识提炼是将具体经验转化为可复用知识的过程，是记忆系统的核心能力。

## 三层记忆架构

### 层次关系

```
情景记忆 → 语义记忆 → 强制规则
   ↑          ↑           ↑
具体事件    通用规律    行为约束
```

### 各层特征

#### 1. 情景记忆（Episodic Memory）
**作用**：记录具体事件和对话的完整过程

**用途**：
- 回溯"当时怎么想的"
- 了解决策背景
- 分析问题根因

**命名**：MMDD-简短描述.md
**归档**：按月份归档

**示例**：
```
0220-记忆系统建设.md
0225-守护进程故障排查.md
```

#### 2. 语义记忆（Semantic Memory）
**作用**：提炼通用规律和知识，可复用

**用途**：
- 不需要每次重新讨论
- 快速检索相关知识
- 跨场景应用

**命名**：[主题]_[类型].md
**类型**：
- 系统架构
- 技术方案
- 工作流程
- 最佳实践
- 方法论

**示例**：
```
系统架构_Hooks事件系统.md
工作流程_系统迭代八步法.md
方法论_知识提炼策略.md
```

#### 3. 强制规则（Forced Rules）
**作用**：系统必须遵守的行为约束

**用途**：
- 让 AI 不会重复犯同样的错
- 确保系统行为一致性
- 保护核心原则

**命名**：强制规则_[领域].md
**优先级**：最高，不可违反

**示例**：
```
强制规则_智能存储路由.md
强制规则_任务与想法追踪.md
```

## 知识提炼流程

### 触发时机

1. **每次自我迭代时**
   - 完成八步法的"提炼"步骤
   - 自动识别可复用模式

2. **发现重复模式时（≥3次）**
   - 同类问题出现3次以上
   - 说明存在通用规律

3. **完成重大项目后**
   - 系统架构变更
   - 功能模块开发
   - 问题解决过程

### 提炼步骤

```
情景记忆
    ↓
识别模式（这是什么类型的知识？）
    ↓
抽象规律（去除具体细节，保留通用方法）
    ↓
验证适用性（能否在其他场景复用？）
    ↓
选择层次（情景记忆/语义记忆/强制规则）
    ↓
编写文档
    ↓
验证质量
```

### 决策树

```
是否涉及系统必须遵守的约束？
├─ 是 → 强制规则
└─ 否 → 是否是通用可复用的知识？
    ├─ 是 → 语义记忆
    └─ 否 → 情景记忆
```

## 提炼模板

### 情景记忆模板

```markdown
# [日期]-[事件简述]

## 背景
- 时间：YYYY-MM-DD
- 触发：[什么事件触发的]
- 目标：[想要达成什么]

## 过程
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 决策
- 关键决策1：[为什么这样选]
- 关键决策2：[为什么这样选]

## 结果
- 成功：[什么做得好]
- 问题：[遇到了什么问题]
- 数据：[量化结果]

## 反思
- 如果重来，会怎么改进？
- 有什么可以复用的经验？
```

## 实践案例

### 案例1：从情景记忆到语义记忆

**情景记忆**：0220-记忆系统建设.md
- 记录了今天创建语义记忆的完整过程
- 包括：检查状态、创建文档、验证功能

**提炼模式**：
- 发现：这是一个重复性工作
- 抽象：知识提炼有其方法论
- 适用：所有需要知识管理的场景

**语义记忆**：方法论_知识提炼策略.md（本文档）
- 通用方法，可复用
- 跨场景应用

### 案例2：从语义记忆到强制规则

**语义记忆**：最佳实践_文件存储策略.md
- 记录了文件管理的最佳实践
- 建议：所有文件保存在 AI_Roland/ 目录

- 用户可能要求保存到外部
- 需要强制执行，不可违反

**强制规则**：强制规则_智能存储路由.md
- 宪法级别规则
- 任何情况不得违反

## 质量标准

### 情景记忆质量检查
- [ ] 时间线清晰
- [ ] 关键决策记录
- [ ] 结果和反思完整

### 语义记忆质量检查
- [ ] 概念定义明确
- [ ] 方法可复用
- [ ] 跨场景适用
- [ ] 有实践案例

### 强制规则质量检查
- [ ] 规则清晰无歧义
- [ ] 触发机制明确
- [ ] 例外情况说明
- [ ] 违反后果清楚

## 知识复用

### 如何使用语义记忆

**场景1：遇到类似问题**
```python
# 搜索相关语义记忆
results = search.search("守护进程启动失败", top_k=3)

# 查看语义记忆
# → 系统架构_守护进程模式.md
# 快速找到解决方案
```

**场景2：系统设计**
```
需要添加新功能
    ↓
搜索类似功能的实现
    ↓
参考语义记忆中的架构和模式
    ↓
复用已验证的方案
```

**场景3：问题排查**
```
遇到错误
    ↓
搜索错误类型
    ↓
查看强制规则
    ↓
识别是否违反规则
```

## 持续改进

### 复用统计
- 记录每个语义记忆的复用次数
- 高复用 → 核心知识，重点维护
- 低复用 → 可能需要优化或删除

### 更新机制
- 发现新的应用场景 → 更新文档
- 发现错误或不足 → 立即修正
- 更好的实现方式 → 替换内容

### 版本管理
```markdown
## 更新历史
- 2026-02-20: 初始版本
- 2026-02-25: 添加案例2
- 2026-03-01: 重构结构，更清晰
```

## 注意事项

⚠️ **不要过度提炼**：不是所有经验都需要提炼
⚠️ **保持实用性**：没有实践案例的语义记忆价值有限
⚠️ **及时更新**：过期的知识比没有知识更危险
⚠️ **质量优先**：1个高质量的语义记忆 > 10个低质量的
